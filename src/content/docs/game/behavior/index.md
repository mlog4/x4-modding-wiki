## Core abstractions

- **Order** — assigned to ships, drives NPC behavior (combat, trade, mining, navigation).
- **Mission** — player-facing tasks generated by the mission framework.
- **Conversation** — NPC dialogue tree.
- **Boarding operation** — boarding state machine attached to a target ship.

## Common questions (planned)

- *"How do I assign a default order to a ship?"* → Order → Actions → `<create_order default=true>`.
- *"How do I add an option to NPC conversation?"* → Conversation → Actions → `<add_player_choice>` / `<add_player_choice_sub>`.
- *"How do I detect when player accepts mission X?"* → Mission → Events → mission framework signals.
- *"How do I modify boarding resistance dynamically?"* → Boarding operation → Gotchas (caching issue).
