This is a **prototype** for evaluating the new site structure. The first version was abandoned because content was too dense, fragmented, and unverified. This version focuses on:

- **API by abstraction** — modders look up the abstractions they need to interact with.
- **Hierarchical browsing** — general containers to subordinate parts.
- **Verified content** — sample-grep checked, working examples.
- **Compact entries** — scannable, Screeps-style schema.

## What's filled in for evaluation

- **[Station](/api/objects/station/)** — the full prototype page, demonstrating the per-page schema.
- **[Shipyard](/api/objects/shipyard/)** — a subtype, demonstrating IS-A relationship.
- **[Module](/api/objects/module/)** — a contained part, demonstrating HAS-A relationship.
- Other categories have **placeholder index pages** to show the navigation structure.

## What to evaluate

1. **Sidebar navigation** — does the hierarchy feel browsable? Is the categorization useful?
2. **Station page** — does it answer common modder questions? Is it the right depth?
3. **Per-page schema** — works as a template for all abstractions?
4. **Cross-references** — links between parent/child/sibling work well?

See [About this prototype](/about/) for the design rationale.
